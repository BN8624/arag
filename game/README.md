# game/ — 모듈식 RPG "임의규칙 자동전투" 정본 구현

> **이 폴더 = 나중에 써먹을 게임(정본).** 측정 하네스(루트의 orchestrator·gates·bank_*·
> obs_run 등)와 **분리**한다. 여기엔 *사람이 직접 짠 신뢰 레퍼런스 구현*만 둔다.

## 두 가지 역할
1. **골든 오라클의 출처** — 이 레퍼런스로 고정 seed의 정확한 전투 결과(트레이스)를 뽑아,
   조립 카드(모델이 만든 구현)가 *이것과 일치*하는지 채점한다(결정19 — 통합 frontier의 강한 오라클).
2. **재사용 게임 정본** — 모듈을 쌓아 실제 RPG 한 판이 돈다. 껍데기(에셋·UI)는 나중에 어댑터로.

## 규칙·계약
[PAMPHLET.md](PAMPHLET.md) — 세계 규칙(신규 임의), 데이터 계약, 모듈 지도, AI 정책.

## 모듈
- `entities.py` — Entity/Status 데이터 계약 + 생성.
- `status.py` — 상태이상 엔진(화상·빙결·중독·감전 + 상호작용 매트릭스).
- `skills.py` — 스킬/콤보(점화·폭발·준비·연계타).
- `combat.py` — 템포 턴제 전투 리졸버 + run_battle.
- `main.py` — seed로 파티 구성 → run_battle → 결정적 리포트/트레이스 출력.

## 분리 원칙
- 모델은 카드 goal만 본다 → 이 레퍼런스 코드는 모델에 안 보인다(누설 금지).
- 측정 카드(bank_cards_rpg.py 등)는 루트에 둔다. 이 폴더는 *정답지*다.
